# 🍬 WowSmak — Инструкция по запуску

Пошаговое руководство, как запустить твой Telegram-магазин конфет с нуля.
Весь интерфейс магазина — на украинском языке. Стек: **Node.js + grammY + Supabase + Cloudinary + GitHub + Render**.

> ⏱️ Времени на всё про всё — примерно 30–40 минут. Программировать не нужно, только копировать ключи и нажимать кнопки.

---

## 📋 Что тебе понадобится (бесплатные аккаунты)

1. **Telegram** — у тебя уже есть
2. **Supabase** — база данных → https://supabase.com
3. **Cloudinary** — хранение фото/видео товаров → https://cloudinary.com
4. **GitHub** — хранение кода → https://github.com
5. **Render** — хостинг бота → https://render.com

Все сервисы имеют бесплатные тарифы, которых хватит для старта.

---

## Шаг 1. Создаём бота в Telegram

1. Открой в Telegram **@BotFather**
2. Отправь команду `/newbot`
3. Придумай имя и username (должен заканчиваться на `bot`)
4. BotFather пришлёт **токен** вида `123456:ABC-DEF...` → **сохрани его**

### Узнать свой Telegram ID (нужен для админки)
- Открой **@userinfobot** и нажми Start → он покажет твой `id` (число). **Сохрани** — это `ADMIN_CHAT_ID`.

---

## Шаг 2. Supabase (база данных)

1. Зарегистрируйся на https://supabase.com → **New project**
2. Придумай пароль к БД, выбери регион поближе (Frankfurt) → Create
3. Когда проект создан, открой **SQL Editor** → New query
4. Скопируй ВЕСЬ текст из файла `supabase/schema.sql` → вставь → **Run**
   - Это создаст все таблицы (товары, категории, заказы, корзина, клиенты, сообщения) и добавит примеры товаров.
5. Зайди в **Project Settings → API** и сохрани:
   - **Project URL** → это `SUPABASE_URL`
   - **service_role** ключ (раздел Project API keys) → это `SUPABASE_SERVICE_KEY`

> ⚠️ `service_role` — секретный ключ. Он используется только на сервере и никогда не попадает на клиент.

> 🔁 Важно: если позже будешь обновлять схему — выполни также `supabase/admin-migration.sql` (он добавляет админ-поля: закупочную цену, штрих-код, рекомендуемую наценку, лог переписки).

---

## Шаг 3. Cloudinary (фото и видео товаров)

1. Зарегистрируйся на https://cloudinary.com
2. На главной (Dashboard) скопируй:
   - **Cloud Name** → `CLOUDINARY_CLOUD_NAME`
   - **API Key** → `CLOUDINARY_API_KEY`
   - **API Secret** → `CLOUDINARY_API_SECRET`
3. Загружать фото товаров можно прямо в Cloudinary (Media Library), а `public_id` указывать в товаре.

## Шаг 4. Заливаем код на GitHub

1. Распакуй этот архив `candy-bot.zip` на компьютере
2. Создай новый репозиторий на https://github.com (можно приватный)
3. Залей файлы. Самый простой способ без терминала:
   - На странице репозитория → **Add file → Upload files** → перетащи все файлы и папки проекта → Commit
   - (Папку `node_modules` заливать НЕ нужно — её и нет в архиве.)

Если умеешь работать с git в терминале:
```bash
cd candy-bot
git init
git add .
git commit -m "WowSmak telegram shop"
git branch -M main
git remote add origin https://github.com/ТВОЙ_ЛОГИН/candy-bot.git
git push -u origin main
```

## Шаг 5. Хостинг на Render

1. Зарегистрируйся на https://render.com через GitHub
2. **New → Web Service** → выбери свой репозиторий `candy-bot`
3. Настройки:
   - **Environment:** Node
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Instance Type:** Free
4. Прокрути до **Environment Variables** и добавь все переменные (см. таблицу ниже)
5. Нажми **Create Web Service** и дождись деплоя
6. Render выдаст адрес вида `https://candy-bot-xxxx.onrender.com` — **скопируй его** и впиши в переменную `PUBLIC_URL`, затем сохрани (сервис передеплоится).

> 💡 В проекте есть `render.yaml` — Render может подхватить настройки автоматически.

### Переменные окружения (Environment Variables)

| Переменная | Где взять / что вписать |
|---|---|
| `BOT_TOKEN` | токен от @BotFather (Шаг 1) |
| `WEBHOOK_SECRET` | придумай любую секретную строку (например `wow-secret-2026`) |
| `PUBLIC_URL` | адрес сервиса на Render (Шаг 5.6) |
| `ADMIN_CHAT_ID` | твой Telegram id (Шаг 1). Несколько админов — через запятую |
| `SUPABASE_URL` | из Supabase (Шаг 2.5) |
| `SUPABASE_SERVICE_KEY` | service_role из Supabase (Шаг 2.5) |
| `CLOUDINARY_CLOUD_NAME` | из Cloudinary (Шаг 3) |
| `CLOUDINARY_API_KEY` | из Cloudinary (Шаг 3) |
| `CLOUDINARY_API_SECRET` | из Cloudinary (Шаг 3) |
| `PORT` | `3000` |

Шаблон всех переменных лежит в файле `.env.example`.

## Шаг 6. Проверка

1. Открой своего бота в Telegram → нажми **Start**
2. Должно появиться меню: `🍬 Каталог`, `🛒 Кошик`, `📦 Мої замовлення`, `ℹ️ Допомога`
3. У тебя (как у админа) дополнительно будет кнопка **🛠 Адмінка**
4. Кнопка меню рядом со скрепкой открывает **Mini App «🍬 Магазин»** (витрина WowSmak)

> ⚠️ На бесплатном тарифе Render сервис «засыпает» после простоя — первый запуск после паузы может занять 30–60 секунд. Это нормально.

## 🛠 Как пользоваться админкой (прямо в боте)

Нажми **🛠 Адмінка** — оттуда можно без программирования:

- **Товары:** добавлять, редактировать любое поле, удалять
- **Цены:** закупочная (видна только админу), продажная, акционная
- **Прибыль и наценка:** бот сам считает прибыль, % наценки и маржу
- **Калькулятор наценки:** кнопки 10 / 20 / 30 … 100 % + рекомендуемая наценка
- **Поштучно:** если в упаковке, например, 1 кг = 160 шт — бот покажет цену за штуку
- **Штрих-код и остаток** товара
- **Рассылка** всем клиентам
- **Переписка с клиентами** (ответ прямо из бота)

## ❓ Если что-то не работает

- **Бот не отвечает** → проверь `BOT_TOKEN` и что сервис на Render задеплоился (вкладка Logs)
- **Не открывается Mini App** → проверь, что `PUBLIC_URL` совпадает с адресом Render и начинается с `https://`
- **Нет товаров** → выполнил ли ты `schema.sql` в Supabase? Можно ещё запустить `npm run seed`
- **Нет кнопки Адмінка** → проверь, что твой Telegram id вписан в `ADMIN_CHAT_ID`
- **Картинки не грузятся** → проверь ключи Cloudinary и `public_id` у товара
